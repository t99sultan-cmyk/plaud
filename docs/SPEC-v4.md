## 2. Модель монетизации

### Базовый тариф
**1000 ₸ = 100 минут** (10 ₸/мин). Это якорная цена.

### Стартовый бонус
**30 минут бесплатно при регистрации** — достаточно для оценки качества (3-5 коротких записей или одна короткая встреча).

### Пакеты кредитов (one-time покупка, не подписка)

| Пакет | Цена | Минут | Цена за минуту | Скидка |
|---|---|---|---|---|
| **Старт** | 1 000 ₸ | 100 | 10.0 ₸ | — |
| **Mini** | 4 990 ₸ | 600 | 8.3 ₸ | 17% |
| **Pro** | 14 990 ₸ | 2 000 | 7.5 ₸ | 25% |
| **Команда** | 49 990 ₸ | 7 000 | 7.1 ₸ | 29% |

Кредиты не сгорают (или сгорают через 12 месяцев — обсудить).

### Себестоимость для нас (за 100 минут)
- AssemblyAI universal-2: ~$0.62 = ~340 ₸
- Claude Sonnet 4.6 (саммари + ~10 вопросов с кешем): ~$0.20 = ~110 ₸
- Storage Supabase (~100 MB на месяц): ~$0.001 = ~0.5 ₸
- Vercel + Inngest на этом объёме: пренебрежимо
- **Итого: ~450 ₸ за 100 мин = 45% маржа на 1000 ₸**

При покупке Pro-пакета (2000 мин за 14990 ₸):
- Себестоимость: ~9000 ₸
- Маржа: ~40%

### Что не делаем
- Подписочную модель (повышает churn, юзеры не любят рекуррентные списания)
- Тариф «Безлимит» (риск злоупотребления)
- Платный тариф «Команда» с кучей пользователей сразу (отдельная сложность с шарингом)

## 3. Промокоды

### Концепция
Админ создаёт коды в `/admin/promocodes`. Юзер вводит код:
- При регистрации (поле «Промокод» опционально на signup форме)
- В разделе «Биллинг / Тарифы» в любой момент

### Типы промокодов
1. **Бесплатные минуты** — `+N минут к балансу`
2. **Скидка %** на следующую покупку пакета
3. **Бесплатный пакет** — например, бесплатный Pro-пакет
4. **Партнёрский** (для лидеров мнений) — даёт автору % с покупок по коду

### Схема БД

```sql
-- Промокоды
create table public.promocodes (
  id              uuid primary key default gen_random_uuid(),
  code            text unique not null,
  description     text,
  type            text not null check (type in ('free_minutes', 'discount_percent', 'free_package')),
  free_minutes    int,            -- если type=free_minutes
  discount_percent int,           -- если type=discount_percent (1-100)
  package_id      text,           -- если type=free_package ('mini', 'pro', 'team')
  max_uses        int,            -- null = безлимит
  used_count      int not null default 0,
  expires_at      timestamptz,
  created_by      uuid references auth.users(id),
  created_at      timestamptz not null default now()
);
create index promocodes_code_idx on public.promocodes(code);

-- История применений
create table public.promocode_redemptions (
  id            uuid primary key default gen_random_uuid(),
  promocode_id  uuid not null references public.promocodes(id) on delete cascade,
  user_id       uuid not null references auth.users(id) on delete cascade,
  redeemed_at   timestamptz not null default now(),
  unique (promocode_id, user_id)  -- один код один раз на юзера
);

-- Баланс минут на юзера
create table public.user_credits (
  user_id       uuid primary key references auth.users(id) on delete cascade,
  free_minutes_remaining  int not null default 30,    -- стартовый бонус
  paid_minutes_remaining  int not null default 0,
  bonus_minutes_remaining int not null default 0,    -- от промокодов
  total_minutes_used      int not null default 0,
  updated_at              timestamptz not null default now()
);
```

### UI для админа: `/admin/promocodes`
- Таблица: Код | Тип | Значение | Использовано / Лимит | Истекает | Создан
- Кнопка «+ Создать промокод» → модалка с полями:
  - Код (можно сгенерировать кнопкой или ввести руками: `LAUNCH50`)
  - Тип (radio: бесплатные минуты / скидка % / бесплатный пакет)
  - Значение (динамически: число минут / процент / выбор пакета)
  - Макс. использований (опционально)
  - Срок действия (опционально)
  - Описание (для админа, не показывается юзеру)
- Кнопка «Деактивировать» (set used_count = max_uses)

### UI для юзера: компонент применения кода
- На signup: поле «Промокод» под формой (опционально)
- В Settings → «Тарифы и баланс»: input + кнопка «Применить»
- На /pricing при покупке пакета: «Есть промокод?» → раскрывается input
- Тосты: «✓ Применено: +500 минут» или «Код истёк / уже использован»

### Server actions
```ts
applyPromocode(code: string): Promise<{ ok: true, granted: string } | { error: string }>
createPromocode(input: PromocodeInput): Promise<{ id: string }>  // admin only
deactivatePromocode(id: string): Promise<{ ok: true }>
```

## 5. История чата — улучшения

### Что есть
- Один чат на запись на юзера
- Сообщения сохраняются в `messages` (verified в коде)
- При повторном открытии записи — история восстанавливается

### Что улучшить

**A) Глобальный список чатов в сайдбаре или отдельной странице**
- Когда у юзера 50 записей — сложно найти где он что-то спрашивал
- Решение: страница `/dashboard/chats` — таблица всех чатов с превью первого вопроса юзера, временем последней реплики, названием записи
- В сайдбаре: пункт «Чаты» с count бэйджем

**B) Поиск по чату**
- На странице записи во вкладке «Чат» — input «Найти в чате»
- Хайлайтит совпадения в сообщениях

**C) Экспорт чата**
- Кнопка «Экспорт» → скачать `.md` с историей разговора

**D) Несколько чатов на одну запись**
- Сейчас один. Иногда юзер хочет «отдельный диалог про бюджет» и «отдельный про команду»
- Можно дать кнопку «+ Новый чат» — создаст ещё одну строку в `chats` (схема уже это позволяет, нужно убрать unique index)

## 7. Бриф для разработчика (executable)

### Phase 1 — Биллинг + промокоды (~3 дня)
1. Миграция: `user_credits`, `promocodes`, `promocode_redemptions`
2. Стартовый бонус: триггер `on auth.users insert → user_credits.free_minutes_remaining = 30`
3. Учёт минут: после `summarize-recording` → списать `recording.duration_sec / 60` из `user_credits`
4. Проверка перед загрузкой: если `free + paid + bonus < предполагаемая длина` (предполагаем по размеру файла) → блок аплоад с CTA «Купить пакет»
5. Server actions: `applyPromocode`, `createPromocode`, `deactivatePromocode`
6. UI: `/dashboard/billing` страница с балансом + покупкой пакетов + полем промокода
7. UI: `/admin/promocodes` страница CRUD
8. UI: на signup форме — поле «Промокод» (опционально)

### Phase 2 — Платёжка (~2 дня)
- Интеграция с **Kaspi Pay** (для Казахстана родной)
- Webhook `/api/payments/kaspi/webhook` → начисление `paid_minutes_remaining`
- Альтернативно: ручная оплата по счёту через CloudPayments / Stripe Connect

### Phase 3 — Лендинг переработка (~1 день)
- Новый Hero (см. п.4)
- Use-cases блок с 3 карточками
- Сравнение с Plaud/Otter/Whisper
- Реальный pricing
- Расширенный FAQ
- Финальный CTA с гарантией

### Phase 4 — История чатов (~1 день)
- Страница `/dashboard/chats` со всеми чатами юзера
- Пункт «Чаты» в сайдбаре с count
- Поиск по чату на странице записи
- Кнопка экспорта чата в .md

### Phase 5 — Шлифовка (~1 день)
- Web-push уведомления (через Web Push API)
- Email-уведомление при готовности (если запись > 30 мин)
- Полировка анимаций (framer-motion на ключевых переходах)

**Итого: ~8 рабочих дней**
